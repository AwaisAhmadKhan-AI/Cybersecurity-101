# Interacting With the Filesystem in Linux

# Introduction

So far, we have learned basic Linux commands such as:

* `echo`
* `whoami`

While useful, real Linux usage requires interacting directly with the filesystem.

This includes:

* Navigating directories
* Listing files
* Reading file contents
* Understanding file paths

Since many Linux systems operate without a graphical desktop environment, command-line filesystem navigation becomes an essential skill.

---

# Core Filesystem Commands

| Command | Full Name               | Purpose                     |
| ------- | ----------------------- | --------------------------- |
| `ls`    | listing                 | List files and directories  |
| `cd`    | change directory        | Move between directories    |
| `cat`   | concatenate             | Display file contents       |
| `pwd`   | print working directory | Show current directory path |

---

# Understanding the Linux Filesystem

Linux organizes data into:

* Directories (folders)
* Files

Everything in Linux exists within a hierarchical filesystem structure.

Example:

```text id="7hmw6f"
/home/ubuntu/Documents
```

---

# Listing Files and Directories (`ls`)

The `ls` command displays the contents of the current directory.

---

# Basic Syntax

```bash id="xqpmr8"
ls
```

---

# Example

```bash id="8o3m2r"
tryhackme@linux1:~$ ls
```

### Output

```text id="u1k7sm"
Important Files
My Documents
Notes
Pictures
```

---

# What `ls` Shows

The output may include:

* Directories
* Files
* Hidden items (with special options)

In this example, Linux displays:

* `Important Files`
* `My Documents`
* `Notes`
* `Pictures`

---

# Listing Another Directory Without Entering It

You can list contents directly:

```bash id="h43o17"
ls Pictures
```

This allows quick inspection without changing directories.

---

# Changing Directories (`cd`)

The `cd` command changes the current working directory.

---

# Basic Syntax

```bash id="7e7lzr"
cd directory_name
```

---

# Example

```bash id="7qdrxh"
cd Pictures
```

---

# Listing Contents After Navigation

```bash id="wkr4j8"
tryhackme@linux1:~/Pictures$ ls
```

### Output

```text id="3om3j9"
dog_picture1.jpg
dog_picture2.jpg
dog_picture3.jpg
dog_picture4.jpg
```

---

# Understanding the Prompt Change

Notice:

```bash id="e8wffj"
~/Pictures
```

This indicates:

* Current directory is now `Pictures`

Linux prompts often display the active directory.

---

# Outputting File Contents (`cat`)

# Printing the Working Directory (`pwd`)

The `pwd` command shows the full path of the current directory.

`pwd` stands for:

> print working directory

---

# Basic Syntax

```bash id="h1d7g2"
pwd
```

---

# Example

```bash id="6yo5w2"
tryhackme@linux1:~/Documents$ pwd
```

### Output

```text id="d6n11n"
/home/ubuntu/Documents
```

---

# Understanding Full Paths

The output:

```text id="vjlwmq"
/home/ubuntu/Documents
```

means:

* `home` → user home directory area
* `ubuntu` → username
* `Documents` → current folder

This is called an:

> Absolute Path

---

# Why `pwd` Matters

When navigating large systems, it is easy to lose track of location.

`pwd` helps:

* Confirm current location
* Avoid mistakes
* Build accurate file paths

---

# Returning to a Directory Using Absolute Paths

Example:

```bash id="jv4z6t"
cd /home/ubuntu/Documents
```

This works regardless of your current location.

---

# Relative vs Absolute Paths

| Type          | Example                           | Meaning                   |
| ------------- | --------------------------------- | ------------------------- |
| Relative Path | `Documents/file.txt`              | Based on current location |
| Absolute Path | `/home/ubuntu/Documents/file.txt` | Full filesystem path      |
