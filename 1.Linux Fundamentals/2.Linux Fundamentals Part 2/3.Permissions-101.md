# Linux Permissions 101

# Overview

Linux permissions control:

* Who can access files
* What actions users can perform
* How secure files and directories are

Permissions are a foundational Linux security concept and are heavily used in:

* System administration
* Cybersecurity
* Web hosting
* Multi-user environments

Linux permissions help enforce:

* Confidentiality
* Integrity
* Access control

---

# Viewing File Permissions

Permissions can be viewed using:

```bash id="n8w3yt"
ls -l
```

or

```bash id="q7v2rm"
ls -lh
```

---

# Example Output

```bash id="x1m9qp"
tryhackme@linux2:~$ ls -lh
```

### Output

```text id="r3t8vc"
-rw-r--r-- 1 cmnatic cmnatic 0 Feb 19 10:37 file1
-rw-r--r-- 8 cmnatic cmnatic 0 Feb 19 10:37 file2
```

---

# Understanding the Permission String

Example:

```text id="d8k4nz"
-rw-r--r--
```

This string contains important permission information.

---

# Structure of Permission Strings

```text id="s6m2wy"
-rwxrwxrwx
```

Broken into sections:

| Section           | Meaning            |
| ----------------- | ------------------ |
| First character   | File type          |
| Next 3 characters | Owner permissions  |
| Next 3 characters | Group permissions  |
| Last 3 characters | Others permissions |

---

# File Type Character

| Symbol | Meaning       |
| ------ | ------------- |
| `-`    | Regular file  |
| `d`    | Directory     |
| `l`    | Symbolic link |

Example:

```text id="p4t8kl"
-rw-r--r--
```

The `-` indicates:

> Regular file

---

# Permission Types

Linux permissions are based on three actions:

| Permission | Symbol | Meaning            |
| ---------- | ------ | ------------------ |
| Read       | `r`    | View file contents |
| Write      | `w`    | Modify file        |
| Execute    | `x`    | Run file/program   |

---

# Permission Groups

Permissions apply to:

| Group  | Description                 |
| ------ | --------------------------- |
| Owner  | User who owns the file      |
| Group  | Users in the assigned group |
| Others | Everyone else               |

---

# Example Breakdown

```text id="m5v2xp"
-rw-r--r--
```

| Section | Meaning              |
| ------- | -------------------- |
| `rw-`   | Owner can read/write |
| `r--`   | Group can read only  |
| `r--`   | Others can read only |

---

# Users vs Groups

Linux supports:

* Individual user permissions
* Shared group permissions

This allows fine-grained access control.

---

# Real-World Example

A web server may need:

* Read/write access to website files

Website customers may:

* Upload files
* Access only their own data

Groups help separate access securely.

---

# Switching Between Users with `su`

Linux allows switching users using:

```bash id="y3k8tw"
su
```

---

# Requirements for Switching Users

To switch users, you typically need:

1. Username
2. User password

---

# Basic `su` Example

```bash id="a7m4rd"
su user2
```

### Example Output

```text id="g6q2lh"
Password:
user2@linux2:/home/tryhackme$
```

---

# Important Observation

Without additional flags:

* You remain in the previous user's directory

---

# Using `su -l`

The `-l` flag means:

> login shell

This loads:

* User environment
* Variables
* Home directory

---

# Example

```bash id="u8x5kc"
su -l user2
```

### Output

```text id="n3v8js"
Password:
user2@linux2:~$
```

---

# Verifying Current Directory

```bash id="c4t7qm"
pwd
```

### Output


---

# Why `-l` Matters

Using `su -l` provides:

* Cleaner environment
* Proper user session
* Correct user paths

This better simulates:

> Actual user login

# Numeric Permissions in Linux

Linux permissions also use:

> Numeric notation

Example:

```text id="f9w3rd"
777
755
644
```

---

# Permission Values

| Permission | Symbol | Value |
| ---------- | ------ | ----- |
| Read       | `r`    | 4     |
| Write      | `w`    | 2     |
| Execute    | `x`    | 1     |

---

# Calculating Numeric Permissions

Permissions are calculated by adding values.

---

# Example: `rwx`

```text id="j8p1yv"
rwx = 4 + 2 + 1 = 7
```

---

# Example: `rw-`

```text id="v4x9md"
rw- = 4 + 2 = 6
```

---

# Example: `r--`

```text id="t7q3ls"
r-- = 4
```

---

# Full Example: `rwxrwxrwx`

| Group  | Permissions | Value |
| ------ | ----------- | ----- |
| Owner  | rwx         | 7     |
| Group  | rwx         | 7     |
| Others | rwx         | 7     |

Result:

```text id="w5n2ke"
777
```

---

# Common Permission Examples

| Symbolic    | Numeric | Meaning                                |
| ----------- | ------- | -------------------------------------- |
| `rwxr-xr-x` | `755`   | Owner full access, others read/execute |
| `rw-r--r--` | `644`   | Owner read/write, others read          |
| `rwx------` | `700`   | Only owner has access                  |

---

# Using `chmod`

Permissions are modified using:

```bash id="m7v1xa"
chmod
```

---

# Example

```bash id="p2r8wu"
chmod 755 script.sh
```

---

# What `755` Means

| Group  | Permission |
| ------ | ---------- |
| Owner  | rwx        |
| Group  | r-x        |
| Others | r-x        |

---

# Another Example

```bash id="y1v6pd"
chmod 750 system_overview.txt
```

---

# Meaning of `750`

| Group  | Permission     |
| ------ | -------------- |
| Owner  | Full access    |
| Group  | Read + execute |
| Others | No access      |

---

# Why Permissions Matter in Cybersecurity

Permissions protect:

* Sensitive files
* User data
* Credentials
* System configurations

Improper permissions may lead to:

* Unauthorized access
* Privilege escalation
* Data exposure

---

# Dangerous Permission Example

```text id="u3m9rv"
777
```

This gives:

* Everyone full access

Often considered:

> Insecure

---

# Secure Permission Practices

| File Type         | Recommended Permissions |
| ----------------- | ----------------------- |
| Scripts           | `755`                   |
| Sensitive configs | `600`                   |
| Private SSH keys  | `600`                   |
| Public web files  | `644`                   |

---

# Common Beginner Mistakes

| Mistake                       | Risk                |
| ----------------------------- | ------------------- |
| Using `777` everywhere        | Major security risk |
| Forgetting execute permission | Script won't run    |
| Incorrect ownership           | Access issues       |
| Overly permissive configs     | Data exposure       |
