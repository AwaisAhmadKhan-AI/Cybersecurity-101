# Common Linux Directories

# Overview

Linux uses a structured filesystem where everything starts from the root directory:

```text id="rootfs"
/
```

Inside `/`, there are standard system directories that serve specific purposes.

Understanding these directories is critical for:

* System navigation
* Security auditing
* Incident response
* Penetration testing
* Linux administration

---

# `/etc` — System Configuration Directory

## Purpose

The `/etc` directory (short for *etcetera*) stores:

* System configuration files
* User account information
* Security settings
* Service configurations

It is one of the most important directories in Linux.

---

## Key Files in `/etc`

| File         | Purpose                             |
| ------------ | ----------------------------------- |
| `passwd`     | Stores user account information     |
| `shadow`     | Stores encrypted password hashes    |
| `sudoers`    | Controls sudo permissions           |
| `sudoers.d/` | Additional sudo configuration files |

---

## Example Listing

```bash id="a1k9qp"
tryhackme@linux2:/etc$ ls
```

### Output

```text id="c4m7xz"
shadow passwd sudoers sudoers.d
```

---

## Important Files Explained

### `/etc/passwd`

Contains:

* Username
* User ID (UID)
* Group ID (GID)
* Home directory
* Login shell

---

### `/etc/shadow`

Contains:

* Encrypted password hashes
* Password policies
* Password expiry information

Hashing method often used:

```text id="sha512"
SHA-512
```

---

### `/etc/sudoers`

Controls:

* Which users can run commands as root
* Administrative privileges

⚠️ Misconfiguration here can lead to full system compromise.

---

# `/var` — Variable Data Directory

## Purpose

`/var` stands for:

> variable data

It stores data that changes frequently during system operation.

---

## What `/var` Contains

* Logs
* Application data
* Temporary system files
* Cache files
* Databases (in some setups)

---

## Common Subdirectories

| Directory      | Purpose                                    |
| -------------- | ------------------------------------------ |
| `/var/log`     | System and service logs                    |
| `/var/tmp`     | Temporary files (persistent across reboot) |
| `/var/backups` | Backup files                               |
| `/var/opt`     | Optional application data                  |

---

## Example Listing

```bash id="v9k2lm"
tryhackme@linux2:/var$ ls
```

### Output

```text id="p7x3qn"
backups log opt tmp
```

---

## Why `/var` is Important

Used for:

* Monitoring system activity
* Debugging issues
* Investigating security incidents

Example:

* Web server logs in `/var/log`

---

# `/root` — Root User Home Directory

## Purpose

`/root` is the home directory of the:

> root user (superuser)

---

## Key Concept

Unlike regular users:

* Regular users → `/home/username`
* Root user → `/root`

---

## Example

```bash id="k2m9vz"
root@linux2:~# ls
```

### Output

```text id="x8q4rw"
myfile myfolder passwords.xlsx
```

---

## Why `/root` Matters

It contains:

* Administrative files
* System-level scripts
* Sensitive data

It is:

> Highly privileged and sensitive

---

# `/tmp` — Temporary Files Directory

## Purpose

`/tmp` is used for:

> temporary storage of files

---

## Key Characteristics

* Writable by all users
* Used for short-term data
* Automatically cleared on reboot

---

## Example Listing

```bash id="m6q2xt"
root@linux2:/tmp# ls
```

### Output

```text id="z1v9nc"
todelete trash.txt rubbish.bin
```

---

## Why `/tmp` is Important

It is commonly used for:

* Temporary scripts
* Intermediate files
* Exploitation tools during testing

---

## Security Note

Because `/tmp` is writable by all users:

* It can be abused for storing malicious scripts
* Requires monitoring in secure environments

---

# Comparison of Key Directories

| Directory | Purpose                | Sensitivity                   |
| --------- | ---------------------- | ----------------------------- |
| `/etc`    | Configuration files    | Very high                     |
| `/var`    | Variable data and logs | Medium to high                |
| `/root`   | Root user home         | Very high                     |
| `/tmp`    | Temporary files        | Low (but risky due to misuse) |

---

# Why These Directories Matter in Cybersecurity

Security professionals often investigate:

## `/etc`

* Password policies
* Sudo access
* Service configurations

## `/var`

* Logs of attacker activity
* System errors
* Web server logs

## `/root`

* Evidence of admin activity
* Sensitive files

## `/tmp`

* Dropped payloads
* Temporary attacker tools

---

# Key Takeaways

* Linux filesystem is structured and standardized
* `/etc` stores system configuration and security settings
* `/var` stores logs and variable system data
* `/root` is the superuser’s home directory
* `/tmp` is temporary, writable, and volatile
* Each directory plays a unique role in system operation and security
