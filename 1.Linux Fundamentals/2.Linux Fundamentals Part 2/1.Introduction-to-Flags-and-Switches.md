# Introduction to Flags and Switches in Linux

# Overview

Most Linux commands support additional options known as:

* Flags
* Switches
* Arguments

These options modify or extend the default behavior of commands.

Flags are one of the reasons Linux command-line tools are:

* Flexible
* Powerful
* Efficient

Understanding how to use flags is essential for:

* Linux administration
* Cybersecurity
* Automation
* Scripting

---

# What Are Flags and Switches?

Flags are additional options added to commands to change how they operate.

They are usually preceded by:

* A single hyphen (`-`)
* A double hyphen (`--`)

---

# General Syntax

```bash id="f9m2qx"
command -flag
```

or

```bash id="s1n8wr"
command --long-option
```

---

# Default Command Behavior

Without flags, commands execute their:

> Default behavior

Example:

```bash id="a4q8tz"
ls
```

lists visible files and directories only.

---

# Example: Using `ls`

Suppose the current directory contains:

* Visible folders
* Hidden folders

---

# Running `ls`

```bash id="j5w7cv"
tryhackme@linux2:~$ ls
```

### Output

```text id="q8k3yn"
folder1
```

Only visible items appear.

---

# Hidden Files in Linux

Files or folders beginning with:

```text id="7w2hce"
.
```

are hidden by default.

Example:

```text id="4j0f7s"
.hiddenfolder
```

---

# Using the `-a` Flag

The `-a` flag means:

> all

It tells `ls` to display hidden files as well.

---

# Example

```bash id="t9m5pk"
ls -a
```

### Output

```text id="w8z3dy"
.hiddenfolder
folder1
```

---

# Understanding Short vs Long Options

Most Linux commands support:

* Short flags
* Long flags

---

# Example

| Short Option | Long Option | Meaning        |
| ------------ | ----------- | -------------- |
| `-a`         | `--all`     | Show all files |

These perform the same function.

---

# Using `--help`

Most Linux commands provide a built-in help menu.

---

# Syntax

```bash id="r7x1pl"
command --help
```

---

# Example

```bash id="v6k2jr"
ls --help
```

---

# What `--help` Provides

The help menu usually includes:

* Command usage
* Available options
* Flag descriptions
* Examples

---

# Example Output Breakdown

```text id="h2m4dw"
-a, --all
```

means:

* `-a` = short option
* `--all` = long option

Both:

> Show hidden files

---

# Why `--help` is Important

`--help` is extremely useful because:

* Every command supports different options
* You do not need to memorize everything
* Documentation is accessible directly from the terminal

---

# Understanding Manual Pages (`man`)

Linux includes built-in documentation called:

> Manual pages

or:

> man pages

---

# Accessing a Manual Page

Syntax:

```bash id="z3f9mc"
man command
```

---

# Example

```bash id="u5d1xy"
man ls
```

---

# What `man` Displays

Manual pages include:

* Command descriptions
* Syntax
* Available flags
* Detailed explanations
* Usage examples

---

# Example Sections of a Man Page

## NAME

Describes the command.

Example:

```text id="q0r9st"
ls - list directory contents
```

---

## SYNOPSIS

Shows command syntax.

Example:

```text id="m1n3qc"
ls [OPTION]... [FILE]...
```

---

## DESCRIPTION

Explains command behavior and options.

---

# Navigating Manual Pages

| Key          | Action    |
| ------------ | --------- |
| `Arrow Keys` | Scroll    |
| `Space`      | Next page |
| `q`          | Quit      |
| `h`          | Help      |

---

# Difference Between `--help` and `man`

| Feature | `--help`        | `man`                     |
| ------- | --------------- | ------------------------- |
| Length  | Short summary   | Detailed documentation    |
| Purpose | Quick reference | Full manual               |
| Output  | Terminal text   | Interactive manual viewer |

---

# Why Flags Matter in Cybersecurity

Cybersecurity professionals heavily rely on command flags to:

* Filter results
* Increase scan detail
* Automate workflows
* Customize tools

Example:

```bash id="t8v4yr"
nmap -sV target
```

The flag:

```text id="j2w9ep"
-sV
```

changes scan behavior.

---

# Common Linux Flag Examples

| Command | Flag | Purpose             |
| ------- | ---- | ------------------- |
| `ls`    | `-a` | Show hidden files   |
| `ls`    | `-l` | Long listing format |
| `grep`  | `-R` | Recursive search    |
| `cat`   | `-n` | Show line numbers   |

---

# Combining Flags

Some commands allow multiple flags together.

Example:

```bash id="w9m7kp"
ls -la
```

This combines:

* `-l` → long listing
* `-a` → show hidden files

---

# Why Hidden Files Matter

Hidden files often contain:

* Configuration settings
* User preferences
* Sensitive information

Examples:

```text id="x3v8fh"
.bashrc
.ssh
.gitconfig
```

Cybersecurity analysts frequently inspect hidden files during:

* Investigations
* Penetration tests
* Forensics
