## Commands Covered

| Command | Full Name | Purpose |
|---------|-----------|---------|
| `touch` | touch | Create a file |
| `mkdir` | make directory | Create a folder |
| `cp` | copy | Copy a file or folder |
| `mv` | move | Move or rename a file or folder |
| `rm` | remove | Delete a file or folder |
| `file` | file | Show the type of a file |

---

## 1) Creating Files with `touch`

The `touch` command is used to create a **new empty file**.

### Syntax

    touch filename

### Example

    touch note

This creates a blank file called **note**.

If you run:

    ls

you may see:

    folder1 note

This shows that the file `note` now exists.

### Important point

`touch` only creates an **empty file**.  
It does **not** put any text inside it.

If you want to add content to the file, you would use something like:

- `echo`
- `nano`

---

## 2) Creating Folders with `mkdir`

The `mkdir` command is used to create a **directory/folder**.

### Syntax

    mkdir foldername

### Example

    mkdir mydirectory

This creates a folder named **mydirectory**.

If you run:

    ls

you may see:

    folder1 mydirectory note

This means the folder was created successfully.

---

## 3) Removing Files with `rm`

The `rm` command is used to **delete files**.

### Syntax

    rm filename

### Example

    rm note

This removes the file **note**.

If you run `ls` again, the file should no longer appear.

---

## 4) Removing Folders with `rm -R`

If you want to delete a **folder**, you usually need to use `rm` with the `-R` option.

### Syntax

    rm -R foldername

### What does `-R` mean?

`-R` means **recursive**.

This tells Linux to remove:

- the folder itself
- everything inside that folder

### Example

    rm -R mydirectory

This removes the folder **mydirectory** and everything inside it.

---

## 5) Copying Files with `cp`

The `cp` command is used to **copy** a file.

It takes **2 arguments**:

1. the original file
2. the name of the copied file

### Syntax

    cp oldfile newfile

### Example

    cp note note2

This copies the file **note** and creates a new file called **note2**.

So after this:

- `note` still exists
- `note2` is the copied version

If you run:

    ls

you may see:

    folder1 note note2

---

## 6) Moving or Renaming Files with `mv`

The `mv` command is used to:

- move a file or folder to another location
- rename a file or folder

It also takes **2 arguments**.

### Syntax

    mv oldname newname

### Example: Renaming a file

    mv note2 note3

This renames **note2** to **note3**.

After this:

- `note2` no longer exists
- it is now called `note3`

If you run:

    ls

you may see:

    folder1 note note3

---

## `cp` vs `mv`

These two commands look similar, but they do different things.

### `cp`
- creates a **copy**
- original file stays there
- new file is created

#### Example

    cp note note2

Result:

- `note` stays
- `note2` is created

### `mv`
- moves a file/folder or renames it
- does **not** create a second copy

#### Example

    mv note2 note3

Result:

- `note2` becomes `note3`

---

## 7) Determining File Type with `file`

Sometimes a file name does not clearly tell us what type of file it is.

For example:

- `notes.txt` looks like a text file
- but `note` gives no clear hint

That is why Linux has the `file` command.

The `file` command tells you **what kind of file it is**.

### Syntax

    file filename

### Example

    file note

Possible output:

    note: ASCII text

This means the file `note` is a **text file**.

---

## Why `file` Is Useful

A file extension can help, but it is not always reliable.

Some files:

- have no extension
- may be renamed
- may not clearly show what they contain

So instead of guessing, you can use:

    file filename

to check the real file type.
