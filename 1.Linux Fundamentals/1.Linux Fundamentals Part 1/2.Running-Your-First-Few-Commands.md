# Running Your First Few Commands in Linux

# Introduction

One of Linux’s biggest advantages is that it can operate with very low system resources.

Unlike operating systems such as Windows, many Linux systems:

* Do not include a graphical desktop environment
* Operate entirely through the command line
* Rely heavily on terminal interaction

This lightweight design makes Linux ideal for:

* Servers
* Cloud systems
* Cybersecurity environments
* Embedded devices

---

# What is the Terminal?

The terminal is a text-based interface used to interact with the operating system.

Instead of:

* Clicking icons
* Opening graphical menus

you type commands directly into the system.

---

# Example Terminal Prompt

```bash id="d0b1cz"
tryhackme@linux1:~$
```

---

# Terminal Prompt Breakdown

| Component   | Meaning                            |
| ----------- | ---------------------------------- |
| `tryhackme` | Current username                   |
| `linux1`    | Hostname (computer name)           |
| `~`         | Current directory (home directory) |
| `$`         | Standard user prompt               |

---

# Why the Terminal Matters

The Linux terminal is extremely powerful because it allows users to:

* Navigate the filesystem
* Create and edit files
* Run programs
* Automate tasks
* Manage systems efficiently

Most cybersecurity tools are heavily command-line based.

# Your First Linux Commands

# `echo`

The `echo` command outputs text to the terminal.

---

# Basic Syntax

```bash id="p7yq1g"
echo text
```

---

# Example 1 — Single Word

```bash id="j4kp4s"
echo Hello
```

### Output

```text id="6o9dzq"
Hello
```

---

# Example 2 — Multiple Words

```bash id="15wsyk"
echo "Hello Friend!"
```

### Output

```text id="w1g3ib"
Hello Friend!
```

---

# Why Use Quotes?

Quotes are required when text contains:

* Spaces
* Special characters

Without quotes:

* The shell treats separated words as multiple arguments

---

# Common Uses of `echo`

## Display Messages

```bash id="l1nq6m"
echo "System Started"
```

---

## Debugging Scripts

```bash id="3k9m4s"
echo "Checking variables..."
```

---

## Writing Data to Files

```bash id="h3j0m1"
echo "Hello" > file.txt
```

# `whoami`

The `whoami` command displays the current logged-in username.

---

# Basic Syntax

```bash id="9yqk8o"
whoami
```

---

# Example

```bash id="0ij14u"
tryhackme@linux1:~$ whoami
```

### Output

```text id="efj8jz"
tryhackme
```

---

# Why `whoami` is Useful

This command helps identify:

* Which account is active
* User privileges
* Execution context

This becomes especially important in cybersecurity and system administration.

---

# User Privileges in Linux

Linux separates users into different permission levels.

## Standard User

Limited permissions.

Cannot:

* Modify system files
* Install protected software
* Manage system-wide settings

---

## Root User

The highest privilege level.

Root can:

* Access all files
* Modify system configurations
* Manage users
* Control the entire system

---

# Why User Context Matters in Cybersecurity

Attackers and defenders both care about:

* Which account is running commands
* What permissions the account has

Compromising a privileged account often gives:

* Greater system access
* More control over resources

```

```

| --------------- | -------- |
| `echo`          | Command  |

---

# Important Linux Concepts

# Case Sensitivity

Linux is case-sensitive.

Example:

```bash id="0w8ex5"
hello
Hello
HELLO
```

These are treated as different values.

---

# Commands Must Be Typed Correctly

Example:

```bash id="rj8e90"
whoami
```

is valid, but:

```bash id="n12iwm"
WhoAmI
```

may fail.

---

# Command History

Linux terminals usually store previously entered commands.

You can:

* Press the Up Arrow key
* Reuse previous commands

This improves efficiency.
