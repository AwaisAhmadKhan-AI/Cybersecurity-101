# Searching for Files in Linux

# Introduction

One of Linux’s greatest strengths is efficiency.

As users become more comfortable with Linux, many actions that once required:

* Manual searching
* Repeated navigation
* Opening directories one-by-one

can instead be automated using powerful command-line tools.

Linux provides utilities that allow users to:

* Search for files
* Locate directories
* Filter results
* Search inside file contents

Two of the most important tools for this are:

* `find`
* `grep`

These commands are essential in:

* System administration
* Cybersecurity
* Digital forensics
* Log analysis

---

# Why File Searching Matters

Large Linux systems may contain:

* Thousands of files
* Deep directory structures
* Numerous configuration files

Manually navigating using:

```bash id="s4fwvs"
cd
ls
```

becomes inefficient.

Search utilities automate this process.

---

# The `find` Command

The `find` command searches for:

* Files
* Directories
* Specific filenames
* File patterns

It can search:

* Current directory
* Entire filesystems
* Nested subdirectories

---

# Basic `find` Syntax

```bash id="hj2z7x"
find -name filename
```

---

# Example Directory Structure

```bash id="ll3m8d"
tryhackme@linux1:~$ ls
```

### Output

```text id="prby4o"
Desktop
Documents
Pictures
folder1
```

Directories may contain additional nested files and folders.

---

# Searching for a Specific File

Suppose we are looking for:

```text id="7e2v7w"
passwords.txt
```

but we do not remember its location.

---

# Example Command

```bash id="ys0jz6"
find -name passwords.txt
```

### Output

```text id="d9e7xg"
./folder1/passwords.txt
```

---

# Understanding the Output

The result:

```text id="x1k9d0"
./folder1/passwords.txt
```

means:

* File exists inside:

```text id="73ah31"
folder1
```

The `./` represents:

> Current directory

---

# Wildcards in Linux

Linux supports:

> Wildcards

Wildcards allow flexible searching.

The most common wildcard is:

```text id="ng1ygt"
*
```

which means:

> Match anything

---

# Finding Files by Extension

Suppose we want to locate all:

```text id="i71u6f"
.txt
```

files.

---

# Example Command

```bash id="0v7b8y"
find -name *.txt
```

### Output

```text id="yyx4fc"
./folder1/passwords.txt
./Documents/todo.txt
```

---

# What Happened Here?

`find` searched:

* Current directory
* All subdirectories

and identified every file ending with:

```text id="u9t8sp"
.txt
```

---

# Common `find` Use Cases

| Goal                | Example                    |
| ------------------- | -------------------------- |
| Find specific file  | `find -name passwords.txt` |
| Find all text files | `find -name *.txt`         |
| Find images         | `find -name *.jpg`         |
| Find logs           | `find -name *.log`         |

# The `grep` Command

The `grep` command searches inside files for:

* Words
* Strings
* Patterns
* Values

Unlike `find`, which searches for filenames, `grep` searches:

> File contents

---

# Basic `grep` Syntax

```bash id="98evqb"
grep "search_term" filename
```

---

# Example Scenario

Suppose a web server log contains:

```text id="rsy6bp"
244 entries
```

Reading the entire file manually is inefficient.

---

# Counting File Lines with `wc`

Example:

```bash id="c6vq4u"
wc -l access.log
```

### Output

```text id="njlwmv"
244 access.log
```

---

# What `wc -l` Means

| Component | Purpose            |
| --------- | ------------------ |
| `wc`      | Word count utility |
| `-l`      | Count lines        |

This shows:

* Total number of log entries

---

# Searching Logs with `grep`

Suppose we want to find activity from IP:

```text id="8v5v3i"
81.143.211.90
```

---

# Example Command

```bash id="0c7vq9"
grep "81.143.211.90" access.log
```

### Output

```text id="zjlwm4"
81.143.211.90 - - [25/Mar/2021:11:17 +0000] "GET / HTTP/1.1" 200 417
```

---

# What `grep` Did

`grep`:

* Searched entire file
* Located matching entries
* Displayed matching lines only

This dramatically improves efficiency.

---

# Why `grep` is Important

Security professionals use `grep` to search:

* Authentication logs
* Web logs
* Configuration files
* Malware indicators
* Error logs

---

# Recursive Searching with `grep`

Sometimes the target data exists across:

* Multiple files
* Multiple directories

`grep` supports recursive searching using:

```text id="3v9b4w"
-R
```

---

# Recursive Search Syntax

```bash id="dcvq9m"
grep -R "search_term" directory
```

---

# Example

```bash id="y4s5q0"
grep -R "PRETTY_NAME" /etc/
```

---

# What This Does

The command:

* Searches every file in:

```text id="7zml4h"
/etc/
```

* Includes all subdirectories
* Displays matching lines

---

# Example Output

```text id="3v9kz0"
grep: /etc/sudoers: Permission denied
/etc/os-release:PRETTY_NAME="Ubuntu"
```

---

# Understanding the Output

## Permission Denied

```text id="4a8x6t"
/etc/sudoers: Permission denied
```

The current user lacks permission to read the file.

---

## Matching Result

```text id="sd1t5v"
/etc/os-release:PRETTY_NAME="Ubuntu"
```

This indicates:

* Match found inside:

```text id="u1m9cc"
/etc/os-release
```

---

# File Paths in `grep` Results

Recursive grep outputs:

```text id="b5h9xn"
filepath:matching_line
```

This helps quickly identify:

* Where matches occur
* Which file contains the data
