# Maintaining Your System: Automation with Cron

# Overview

Linux systems often need to perform repetitive tasks automatically.

Examples include:

* Backing up files
* Running scripts
* Launching applications
* System maintenance
* Scheduled monitoring

Linux handles task automation using:

```text id="7y8jtm"
cron
```

and:

```text id="0bpkdr"
crontabs
```

---

# What is Cron?

`cron` is a background service (daemon) responsible for:

> Running scheduled tasks automatically

It starts during:

```text id="9xqgh1"
System boot
```

and continuously checks for scheduled jobs.

---

# What is a Cron Job?

A:

```text id="pz0u53"
Cron job
```

is:

> A scheduled command or task executed automatically

---

# What is a Crontab?

A:

```text id="3vpx08"
crontab
```

(short for *cron table*) is a special configuration file used by cron.

It contains:

* Scheduling information
* Commands to execute

---

# How Cron Works

1. System boots
2. `cron` service starts
3. `cron` reads crontab files
4. Scheduled commands execute automatically

---

# Crontab Structure

Each cron job contains:

```text id="dh7j0u"
MIN HOUR DOM MON DOW CMD
```

---

# Crontab Fields Explained

| Field  | Meaning            |
| ------ | ------------------ |
| `MIN`  | Minute             |
| `HOUR` | Hour               |
| `DOM`  | Day of month       |
| `MON`  | Month              |
| `DOW`  | Day of week        |
| `CMD`  | Command to execute |

---

# Crontab Format Example

```bash id="d77r6d"
0 */12 * * * cp -R /home/cmnatic/Documents /var/backups/
```

---

# Breakdown of the Example

| Part        | Meaning            |
| ----------- | ------------------ |
| `0`         | Run at minute 0    |
| `*/12`      | Every 12 hours     |
| `*`         | Every day of month |
| `*`         | Every month        |
| `*`         | Every day of week  |
| `cp -R ...` | Backup command     |

---

# What This Cron Job Does

This command:

* Copies the `Documents` directory
* Stores backup in `/var/backups/`
* Runs every 12 hours

---

# Understanding Wildcards (`*`)

The asterisk:

```text id="y3ys0n"
*
```

means:

> Any value

---

# Example Meaning

```bash id="cdd08r"
* * * * *
```

means:

```text id="nnbrb9"
Every minute of every hour of every day
```

---

# Common Cron Expressions

| Expression     | Meaning               |
| -------------- | --------------------- |
| `* * * * *`    | Every minute          |
| `0 * * * *`    | Every hour            |
| `0 0 * * *`    | Every day at midnight |
| `0 */12 * * *` | Every 12 hours        |
| `0 9 * * 1`    | Every Monday at 9 AM  |

---

# Special Characters in Cron

| Symbol | Meaning               |
| ------ | --------------------- |
| `*`    | Wildcard (all values) |
| `,`    | Multiple values       |
| `-`    | Range                 |
| `/`    | Step intervals        |

---

# Example Using Step Values

```bash id="tgbjlwm"
*/5 * * * *
```

means:

```text id="n4os4m"
Every 5 minutes
```

# Editing Crontabs

To edit your cron jobs:

```bash id="9qfp22"
crontab -e
```

---

# What Happens?

Linux:

* Opens a text editor
* Allows modifying cron jobs
* Saves schedule automatically

# Viewing Existing Cron Jobs

Use:

```bash id="f0qyxr"
crontab -l
```

---

# Removing Cron Jobs

Use:

```bash id="kzz6d8"
crontab -r
```

⚠️ Warning:
This removes all cron jobs for the current user.

# Security Risks of Cron Jobs

Poorly configured cron jobs can:

* Execute malicious scripts
* Create privilege escalation opportunities
* Expose sensitive data

---

# Common Security Issues

| Risk                  | Description                        |
| --------------------- | ---------------------------------- |
| Writable scripts      | Attackers modify scheduled scripts |
| Running as root       | High privilege abuse               |
| Weak file permissions | Unauthorized access                |
| Hidden persistence    | Malware auto-starting              |

---

# Important Cron Security Practices

| Best Practice               | Reason                 |
| --------------------------- | ---------------------- |
| Restrict permissions        | Prevent tampering      |
| Avoid unnecessary root jobs | Reduce risk            |
| Audit cron jobs regularly   | Detect malicious tasks |
| Log cron activity           | Improve visibility     |

---

# Cron vs Manual Execution

| Manual Tasks         | Cron Automation      |
| -------------------- | -------------------- |
