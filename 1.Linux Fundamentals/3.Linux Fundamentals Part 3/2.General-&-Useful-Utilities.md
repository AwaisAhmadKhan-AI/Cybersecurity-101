# General & Useful Linux Utilities

# Overview

Linux provides many powerful utilities for:

* Downloading files
* Transferring data between systems
* Hosting files temporarily
* Managing remote systems

These utilities are heavily used in:

* System administration
* Networking
* Cybersecurity
* Penetration testing

This section covers:

* `wget`
* `scp`
* Python HTTP server

---

# Downloading Files with `wget`

# What is `wget`?

`wget` is a command-line utility used to:

> Download files from the web

It supports:

* HTTP
* HTTPS
* FTP

---

# Basic Syntax

```bash id="a1p9zk"
wget URL
```

---

# Example

```bash id="j8s2mv"
wget https://assets.tryhackme.com/additional/linux-fundamentals/part3/myfile.txt
```

---

# What Happens?

`wget`:

1. Connects to the web server
2. Downloads the file
3. Saves it in the current directory

---

# Common `wget` Use Cases

| Use Case         | Example             |
| ---------------- | ------------------- |
| Download scripts | Security tools      |
| Download images  | Media files         |
| Retrieve backups | Archives            |
| Fetch payloads   | Penetration testing |

---

# Why `wget` is Important in Cybersecurity

Security professionals use `wget` to:

* Download enumeration scripts
* Retrieve payloads
* Transfer tools to compromised systems
* Fetch updates or logs

---

# Secure File Transfer with `scp`

# What is `scp`?

`scp` stands for:

> Secure Copy

It uses:

```text id="fw9czr"
SSH (Secure Shell)
```

to securely transfer files between systems.

---

# Key Features of `scp`

| Feature          | Description                    |
| ---------------- | ------------------------------ |
| Encryption       | Data protected during transfer |
| Authentication   | Requires valid credentials     |
| Remote transfers | Works across networks          |

---

# SCP Transfer Models

`scp` supports:

1. Local → Remote
2. Remote → Local

---

# SCP Syntax

```bash id="r5du9j"
scp SOURCE DESTINATION
```

---

# Copying Local File to Remote System

## Scenario

| Variable        | Value             |
| --------------- | ----------------- |
| Remote IP       | `192.168.1.30`    |
| Remote User     | `ubuntu`          |
| Local File      | `important.txt`   |
| Remote Filename | `transferred.txt` |

---

# Command

```bash id="y2pfqt"
scp important.txt ubuntu@192.168.1.30:/home/ubuntu/transferred.txt
```

---

# Breakdown

| Component                      | Meaning                 |
| ------------------------------ | ----------------------- |
| `scp`                          | Secure copy command     |
| `important.txt`                | Local source file       |
| `ubuntu@192.168.1.30`          | Remote user and host    |
| `/home/ubuntu/transferred.txt` | Remote destination path |

---

# Copying Remote File to Local System

## Scenario

| Variable       | Value           |
| -------------- | --------------- |
| Remote IP      | `192.168.1.30`  |
| Remote User    | `ubuntu`        |
| Remote File    | `documents.txt` |
| Local Filename | `notes.txt`     |

---

# Command

```bash id="3bzt93"
scp ubuntu@192.168.1.30:/home/ubuntu/documents.txt notes.txt
```

---

# What Happens?

This command:

* Downloads `documents.txt`
* Saves it locally as `notes.txt`

---

# Why `scp` Matters in Cybersecurity

`scp` is commonly used to:

* Transfer tools to target systems
* Download evidence/logs
* Move scripts between servers
* Backup sensitive files securely

# Serving Files with Python HTTP Server

# What is Python HTTP Server?

Python includes a lightweight built-in web server module:

```text id="7k0rfs"
http.server
```

This allows a system to quickly serve files over HTTP.

---

# Why It’s Useful

Useful for:

* Quick file sharing
* Transferring payloads
* Hosting scripts temporarily
* Lab environments

---

# Starting the Server

## Basic Syntax

```bash id="l3x7np"
python3 -m http.server
```

---

# Example

```bash id="mz2n6j"
tryhackme@linux3:/webserver# python3 -m http.server
```

---

# Output

```text id="snshnh"
Serving HTTP on 0.0.0.0 port 8000
```

---

# Important Default Behavior

The server:

* Serves files from the current directory
* Uses port:

```text id="gdl6pk"
8000
```

by default.

---

# Downloading Files from the Server

If the server IP is:

```text id="y9x5an"
10.113.180.204
```

and the file name is:

```text id="e80bdj"
myfile
```

then use:

```bash id="fjlwm8"
wget http://10.113.180.204:8000/myfile
```

---

# Example Download Output

```text id="4f6c1f"
HTTP request sent, awaiting response... 200 OK
Saving to: ‘file’
```

---

# Important Operational Note

The terminal running:

```bash id="v2tx3y"
python3 -m http.server
```

remains occupied until stopped.

You must:

> Open another terminal to use `wget`

---

# Common Cybersecurity Use Cases

Python HTTP server is heavily used for:

* Payload hosting
* File staging
* Script delivery
* Rapid file transfer in labs

---

# Advantages of Python HTTP Server

| Advantage         | Explanation               |
| ----------------- | ------------------------- |
| Lightweight       | No installation needed    |
| Fast setup        | One command               |
| Built into Python | Available on many systems |
| Useful for labs   | Easy local hosting        |

---

# Limitation of Python HTTP Server

## Major Limitation

You must know:

* Exact file name
* Exact file location

There is:

> No advanced indexing or management

---

# Alternative Mentioned: Updog

`Updog` is a more advanced lightweight file server that offers:

* Better indexing
* Easier navigation
* Improved usability

# Common Beginner Mistakes

| Mistake                      | Result             |
| ---------------------------- | ------------------ |
