# 19. Maintaining Your System: Logs

## Introduction to Logs

Logs are records generated by:

* Operating systems
* Applications
* Services
* Security tools

They provide visibility into:

* System activity
* User actions
* Errors
* Security events
* Network activity

Logs are essential for:

* Monitoring
* Troubleshooting
* Incident response
* Forensics
* Security auditing

---

# Log File Location

Most Linux logs are stored in:

```text id="o2n6z1"
/var/log
```

---

# Why Logs Matter

Logs help administrators:

* Detect attacks
* Troubleshoot issues
* Monitor performance
* Investigate incidents
* Track authentication attempts

Without logs, identifying malicious activity becomes extremely difficult.

---

# Log Rotation

Linux systems automatically manage logs through a process called:

```text id="mx9t1q"
log rotation
```

---

## Purpose of Log Rotation

Prevents logs from:

* Consuming too much disk space
* Becoming too large to manage

Old logs are:

* Archived
* Compressed
* Deleted after a period of time

---

# Common Linux Logs

## Apache2 Web Server Logs

### Access Log

Records every HTTP request made to the web server.

Typical information includes:

* Visitor IP address
* Requested page
* HTTP method
* Response code
* Browser user-agent

---

### Example Access Log Entry

```text id="3ghm8k"
192.168.1.10 - - [25/Mar/2026:10:22:15] "GET /index.html HTTP/1.1" 200
```

---

## Error Log

Records server-side problems.

Examples:

* Missing files
* Server crashes
* Permission issues
* Application errors

---

# Security Logs

## Authentication Logs

Track:

* Login attempts
* Failed passwords
* SSH access
* Privilege escalation

Useful for detecting:

* Brute-force attacks
* Unauthorized access

---

## Fail2Ban Logs

Fail2Ban monitors repeated failed login attempts.

Purpose:

* Detect brute-force attacks
* Automatically block attackers

---

## UFW Firewall Logs

UFW (Uncomplicated Firewall) logs:

* Allowed connections
* Blocked traffic
* Firewall events

Useful for:

* Network monitoring
* Detecting suspicious traffic

---

# Useful Log Files

| Log File                    | Purpose                       |
| --------------------------- | ----------------------------- |
| /var/log/auth.log           | Authentication events         |
| /var/log/syslog             | General system logs           |
| /var/log/apache2/access.log | Web requests                  |
| /var/log/apache2/error.log  | Web server errors             |
| /var/log/ufw.log            | Firewall activity             |
| /var/log/fail2ban.log       | Brute-force protection events |

---

# Viewing Logs

## Using cat

```bash id="m1jx4e"
cat /var/log/syslog
```

Displays entire file contents.

---

## Using less

```bash id="1jlwmn"
less /var/log/syslog
```

Allows scrolling through logs.

---

## Using tail

```bash id="dlj90j"
tail /var/log/syslog
```

Displays last lines of a log.

---

## Real-Time Monitoring

```bash id="a3vz5l"
tail -f /var/log/syslog
```

Continuously watches log updates in real time.

---

# Searching Logs with grep

## Find Failed Login Attempts

```bash id="1l0eqw"
grep "Failed password" /var/log/auth.log
```

---

## Search Apache Logs

```bash id="n8r2aq"
grep "404" /var/log/apache2/access.log
```

Finds failed page requests.

---

# Common Security Investigations

## Detecting Brute Force Attempts

Look for:

* Repeated failed logins
* Multiple attempts from same IP

Example:

```bash id="2w7ylo"
grep "Failed password" /var/log/auth.log
```

---

## Investigating Web Attacks

Review:

* Suspicious URLs
* Unusual request patterns
* Repeated 404 errors
* SQL injection attempts

---

## Monitoring Firewall Activity

Check blocked traffic:

```bash id="htxj1r"
cat /var/log/ufw.log
```

---

# Importance of Logs in Cybersecurity

Logs are critical for:

* Threat detection
* Incident response
* Digital forensics
* Compliance auditing
* Security monitoring

Attackers often attempt to:

* Delete logs
* Modify logs
* Disable logging

Protecting logs is therefore essential.

---

# Best Practices

## Regularly Review Logs

Monitor for:

* Failed logins
* Suspicious IP addresses
* Unexpected errors

---

## Centralize Logs

Use centralized logging systems:

* SIEM tools
* Syslog servers

Examples:

* Splunk
* ELK Stack
* Graylog

---

## Restrict Log Access

Logs may contain:

* Usernames
* IP addresses
* Sensitive system data

Only authorized users should access them.

---

## Enable Log Rotation

Prevent disk exhaustion and maintain system stability.

---

## Monitor Authentication Events

Repeated failures may indicate active attacks.

---

# Key Takeaways

* Logs record system and application activity
* Most Linux logs are stored in /var/log
* Access logs track requests
* Error logs track failures
* Security logs help detect attacks
* grep and tail are essential log analysis tools
* Logs are vital for incident response and forensics
