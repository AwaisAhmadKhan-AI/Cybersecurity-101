# Terminal Text Editors in Linux

# Overview

So far, text files have been created using commands like:

```bash
echo
```

combined with output redirection operators:

```bash
>
>>
```

While useful for quick tasks, this method becomes inefficient for:

* Multi-line files
* Editing existing text
* Managing larger documents
* Writing scripts or code

Linux provides terminal-based text editors to solve this problem.

---

# Why Terminal Text Editors Matter

Terminal text editors allow users to:

* Create files
* Edit files
* Modify configurations
* Write scripts
* Manage logs

These tools are essential for:

* Linux administration
* Cybersecurity
* Software development
* Remote server management

---

# Common Linux Terminal Editors

| Editor | Difficulty        | Features                         |
| ------ | ----------------- | -------------------------------- |
| `nano` | Beginner-friendly | Simple and easy to use           |
| `vim`  | Advanced          | Powerful and highly customizable |

---

# Nano Text Editor

# What is Nano?

Nano is a lightweight terminal text editor designed for:

* Simplicity
* Ease of use
* Quick editing

It is often pre-installed on many Linux distributions.

---

# Opening Nano

Basic syntax:

```bash
nano filename
```

---

# Example

```bash
nano myfile
```

---

# What Happens?

Nano:

* Opens the file if it exists
* Creates the file if it does not exist

---

# Nano Interface Example

```text
GNU nano 4.8                                  myfile

Hello TryHackMe
I can write things into "myfile"
```

---

# Basic Nano Navigation

| Key        | Action      |
| ---------- | ----------- |
| Arrow Keys | Move cursor |
| Enter      | New line    |
| Backspace  | Delete text |

---

# Nano Shortcut Keys

Nano uses:

```text
^
```

to represent:

> Ctrl key

Example:

```text
^X
```

means:

```text
Ctrl + X
```

---

# Common Nano Shortcuts

| Shortcut   | Function              |
| ---------- | --------------------- |
| `Ctrl + X` | Exit Nano             |
| `Ctrl + O` | Save file (Write Out) |
| `Ctrl + W` | Search text           |
| `Ctrl + K` | Cut line              |
| `Ctrl + U` | Paste text            |
| `Ctrl + G` | Help menu             |

---

# Saving a File in Nano

To save:

1. Press:

```text
Ctrl + O
```

2. Press:

```text
Enter
```

to confirm filename.

---

# Exiting Nano

To exit:

```text
Ctrl + X
```

---

Nano is popular because:

* Controls are displayed on-screen
* Easy to learn
* Minimal complexity
* Good for quick edits

# Common Nano Use Cases

Nano is often used for:

* Editing configuration files

# VIM Text Editor

# What is VIM?

VIM is a powerful and advanced terminal text editor.

It is widely used by:

* System administrators
* Developers
* Cybersecurity professionals

---

# Why VIM is Popular

VIM offers:

* High efficiency
* Extensive customization
* Keyboard-driven workflows
* Advanced editing features

---

# Major Benefits of VIM

| Feature             | Benefit                            |
| ------------------- | ---------------------------------- |
| Customization       | Adjustable shortcuts and workflows |
| Syntax Highlighting | Easier code readability            |
| Performance         | Lightweight and fast               |
| Availability        | Installed on many Linux systems    |

---

# Syntax Highlighting

VIM can colorize:

* Programming languages
* Configuration files
* Scripts

This improves:

* Readability
* Error detection
* Development efficiency

---

# Why Cybersecurity Professionals Use VIM

VIM is valuable because:

* It works over SSH
* It is available on many systems
* It performs well in minimal environments

This makes it ideal for:

* Remote server management
* Incident response
* Penetration testing

---

# Nano vs VIM Comparison

| Feature                   | Nano      | VIM                 |
| ------------------------- | --------- | ------------------- |
| Ease of use               | Easy      | Difficult initially |
| Learning curve            | Low       | High                |
| Features                  | Basic     | Advanced            |
| Customization             | Minimal   | Extensive           |
| Popular for beginners     | Yes       | No                  |
| Popular for professionals | Sometimes | Very common         |

# Important VIM Note

VIM has:

> Modes

This is one reason beginners find it difficult.

Examples:

* Insert mode
* Command mode
* Visual mode
