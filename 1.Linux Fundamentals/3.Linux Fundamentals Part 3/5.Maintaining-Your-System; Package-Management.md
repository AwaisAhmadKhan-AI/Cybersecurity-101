# 18. Maintaining Your System: Package Management

## Introduction to Packages and Repositories

Linux systems use package managers to install, update, and remove software efficiently.

In Ubuntu and Debian-based systems, the primary package manager is:

```bash
apt
```

Packages are downloaded from repositories (repos), which are servers containing software packages and updates.

---

## What is a Repository?

A repository is a storage location that contains:

* Software packages
* Updates
* Dependencies
* Security patches

Operating systems maintain official repositories, but users can also add third-party repositories.

---

## Benefits of Package Management

### Centralized Software Management

Install and manage applications from one place.

### Automatic Dependency Handling

Required libraries and dependencies are installed automatically.

### Security Updates

Repositories provide trusted updates and patches.

### Easy Upgrades

Software can be updated using simple commands.

---

# APT Package Manager

## Common APT Commands

| Command     | Purpose                    |
| ----------- | -------------------------- |
| apt update  | Refresh package list       |
| apt upgrade | Upgrade installed packages |
| apt install | Install software           |
| apt remove  | Remove software            |
| apt search  | Search repositories        |

---

## Updating Package Lists

```bash
sudo apt update
```

This command:

* Contacts repositories
* Downloads latest package information
* Updates local package database

---

## Upgrading Installed Software

```bash
sudo apt upgrade
```

Installs newer versions of installed packages.

---

## Installing Software

```bash
sudo apt install package-name
```

Example:

```bash
sudo apt install sublime-text
```

---

## Removing Software

```bash
sudo apt remove package-name
```

Example:

```bash
sudo apt remove sublime-text
```

---

# Software Repositories

## Repository Configuration Files

Repositories are stored in:

```text
/etc/apt/sources.list
```

Additional repositories are commonly stored in:

```text
/etc/apt/sources.list.d/
```

Sometimes software is not available in official repositories.

# GPG Keys and Trust

## What is a GPG Key?

GPG (GNU Privacy Guard) keys verify software authenticity.

Purpose:

* Confirm software integrity
* Verify software source
* Prevent tampering

---

## Adding a GPG Key

Example:

```bash
wget -qO - https://download.sublimetext.com/sublimehq-pub.gpg | sudo apt-key add -
```

### Breakdown

| Part          | Purpose                  |
| ------------- | ------------------------ |
| wget          | Download file            |
| -qO -         | Quiet output to terminal |
| apt-key add - | Add trusted GPG key      |

---

# Adding a Repository Manually

## Step 1: Create Repository File

```bash
sudo nano /etc/apt/sources.list.d/sublime-text.list
```

---

## Step 2: Add Repository Entry

Example repository line:

```text
deb https://download.sublimetext.com/ apt/stable/
```

Save and exit Nano.

---

## Step 3: Update Package Lists

```bash
sudo apt update
```

This tells APT to recognize the newly added repository.

---

## Step 4: Install Software

```bash
sudo apt install sublime-text
```

---

# Removing Repositories

## Remove Repository via Command

```bash
sudo add-apt-repository --remove ppa:PPA_Name/ppa
```

---

## Remove Repository Manually

Delete the repository file:

```bash
sudo rm /etc/apt/sources.list.d/sublime-text.list
```

Then update package lists:

```bash
sudo apt update
```

# Important Security Concepts

## Why Trusted Repositories Matter

Installing software from untrusted repositories can lead to:

* Malware infections
* Backdoors
* Privilege escalation
* System compromise

---

## Best Practices

### Use Official Repositories

Prefer trusted package sources.

### Verify GPG Keys

Always confirm authenticity.

### Keep Systems Updated

Regularly install security patches.

### Remove Unused Repositories

Reduce attack surface.

### Avoid Random PPAs

Third-party repos may contain unsafe software.
