# Processes 101 in Linux

# Overview

A process is:

> A running program on a Linux system

Every application, command, or service running on Linux operates as a process.

Processes are managed by:

```text
The Linux Kernel
```

---

# What is a PID?

Each process receives a unique identifier called a:

```text
PID (Process ID)
```

---

# Key PID Characteristics

* Assigned when the process starts
* Incremented sequentially
* Used to manage or terminate processes

Example:

* Process 60 → PID 60

---

# Viewing Running Processes

# Using `ps`

The `ps` command displays:

* Running processes
* Process IDs
* CPU usage
* Associated terminals
* Executed commands

---

# Basic Syntax

```bash
ps
```

---

# Information Displayed by `ps`

| Field | Meaning              |
| ----- | -------------------- |
| PID   | Process ID           |
| TTY   | Terminal session     |
| TIME  | CPU time used        |
| CMD   | Command/program name |

---

# Example

```bash
ps
```

Example output:

```text
PID TTY          TIME CMD
204 pts/0    00:00:00 ps
```

---

# Important Observation

Each new process receives a new PID.

Example:

* One `ps` command → PID 204
* Next `ps` command → PID 205

---

# Viewing All Processes with `ps aux`

Basic `ps` only shows:

* Current user's processes

To view:

* System processes
* Other users' processes

Use:

```bash
ps aux
```

---

# What `ps aux` Shows

| Component | Meaning                             |
| --------- | ----------------------------------- |
| `a`       | Show processes for all users        |
| `u`       | Display user information            |
| `x`       | Include processes without terminals |

---

# Example Users Seen

```text
root
cmnatic
```

---

# Real-Time Monitoring with `top`

# What is `top`?

`top` displays:

> Live process statistics

Unlike `ps`, which provides a snapshot, `top` updates continuously.

---

# Running `top`

```bash
top
```

---

# Information Displayed by `top`

| Information       | Description            |
| ----------------- | ---------------------- |
| CPU usage         | Processor consumption  |
| Memory usage      | RAM utilization        |
| Running processes | Active tasks           |
| Process priority  | Scheduling information |

---

# Why `top` is Useful

Useful for:

* Troubleshooting
* Resource monitoring
* Performance analysis
* Detecting suspicious activity

---

# Managing Processes

# Killing Processes

Linux allows sending signals to processes.

The most common management command is:

```bash
kill PID
```

---

# Example

```bash
kill 1337
```

This sends a signal to:

```text
PID 1337
```

---

# Common Process Signals

| Signal    | Purpose               |
| --------- | --------------------- |
| `SIGTERM` | Gracefully terminate  |
| `SIGKILL` | Forcefully terminate  |
| `SIGSTOP` | Pause/suspend process |

---

# Signal Details

## SIGTERM

```text
Graceful shutdown
```

Allows:

* Cleanup tasks
* Saving state
* Proper closing

---

## SIGKILL

```text
Immediate termination
```

* Cannot be ignored
* No cleanup occurs

---

## SIGSTOP

```text
Pause process execution
```

Used for:

* Temporary suspension
* Background management

---

# How Processes Start

# Namespaces

Linux uses:

```text
Namespaces
```

to isolate processes and resources.

---

# Resources Controlled by Namespaces

| Resource           | Example                     |
| ------------------ | --------------------------- |
| CPU                | Processing allocation       |
| RAM                | Memory allocation           |
| Process visibility | Isolation between processes |

---

# Why Namespaces Matter

Namespaces improve:

* Security
* Isolation
* Resource management

Processes inside one namespace:

> Cannot necessarily see processes in another namespace

---

# The Init Process

The first process started during boot has:

```text
PID 0
```

On Ubuntu systems, the initialization system is commonly:

```text
systemd
```

---

# What is `systemd`?

`systemd`:

* Starts system services
* Manages processes
* Controls background services

It acts between:

* Kernel
* Users
* Applications

---

# Child Processes

Programs launched after boot become:

```text
Child processes
```

of `systemd`.

This allows:

* Better management
* Easier monitoring
* Organized process control

---

# Starting Services on Boot

Some services should automatically start during boot.

Examples:

* Web servers
* Database servers
* SSH servers

---

# Managing Services with `systemctl`

`systemctl` interacts with:

```text
systemd
```

---

# Basic Syntax

```bash
systemctl [option] [service]
```

---

# Example: Start Apache

```bash
systemctl start apache2
```

---

# Common `systemctl` Options

| Option    | Purpose             |
| --------- | ------------------- |
| `start`   | Start service       |
| `stop`    | Stop service        |
| `enable`  | Start on boot       |
| `disable` | Prevent auto-start  |
| `status`  | View service status |

---

# Example Commands

## Enable Apache on Boot

```bash
systemctl enable apache2
```

---

## Check Apache Status

```bash
systemctl status apache2
```

---

# Foreground vs Background Processes

Processes can run in:

* Foreground
* Background

---

# Foreground Processes

Foreground processes:

* Occupy the terminal
* Display output directly

Example:

```bash
echo "Hi THM"
```

---

# Background Processes

Background processes:

* Run without blocking terminal usage
* Allow multitasking

---

# Using `&` to Background a Process

Example:

```bash
echo "Hi THM" &
```

---

# What Happens?

Instead of output, Linux shows:

* Process ID
* Background job information

---

# Why Backgrounding is Useful

Useful for:

* Long-running tasks
* File transfers
* Scripts
* Servers

---

# Suspending a Process with `Ctrl + Z`

Keyboard shortcut:

```text
Ctrl + Z
```

This:

* Suspends the process
* Moves it to background

---

# Example Use Cases

Useful when:

* Script loops endlessly
* Need temporary pause
* Want terminal control back

---

# Bringing a Process Back to Foreground

Use:

```bash
fg
```

---

# What `fg` Does

`fg`:

* Restores background process
* Returns interaction to terminal

---

# Example Workflow

## Start Script

```bash
./background.sh
```

---

## Suspend Script

```text
Ctrl + Z
```

---

## Resume Script

```bash
fg
```
